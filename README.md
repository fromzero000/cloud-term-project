# 프로젝트 제안서: AWS 기반 택시 동승 매칭 서비스

## 1. 프로젝트 개요
* [cite_start]**주제**: AWS 트래픽 분산 처리를 활용한 대학생 맞춤형 택시 동승(합승) 매칭 및 실시간 위치 공유 웹 서비스 [cite: 33, 37]
* [cite_start]**목적**: 동일한 목적지를 가진 사용자들을 연결하여 택시비를 절감하고, 교내 축제나 시험 기간 등 특정 시간대 트래픽 급증 상황에서 AWS를 활용한 안정적인 클라우드 기반 아키텍처를 구현 [cite: 35, 37, 38, 50, 51]

## 2. 필요성 및 활용방안
* **필요성**:
    * [cite_start]대학생의 경제적 부담 완화 [cite: 46]
    * [cite_start]기존 커뮤니티(에브리타임 등)의 게시판 방식은 특정 이벤트 시 서버 마비나 게시글 매몰 등의 한계가 있음 [cite: 47, 48]
    * [cite_start]약속 장소에서 서로를 찾지 못하는 문제를 해결할 전용 서비스의 부재 [cite: 48, 49]
* **활용방안**:
    * [cite_start]**방 개설 및 참여**: 출발지, 목적지, 시간 조건에 따른 방 생성 및 검색 [cite: 53, 54]
    * [cite_start]**안전**: '동성 간 합승' 정책을 벤치마킹하여 안전한 커뮤니티 구축 [cite: 55, 69, 71]
    * [cite_start]**실시간 위치 공유**: Kakao Map API를 연동하여 매칭 완료 후 탑승지에서 서로의 실시간 위치 확인 [cite: 56, 80, 81]
    * [cite_start]**정산**: 예상 요금 1/N 계산 및 송금 링크 제공 [cite: 60, 73]

## 3. 선행기술 조사 및 차별성
* [cite_start]**반반택시(코나투스)**: 합법적 동승 서비스, 동성 매칭 [cite: 66, 67, 69]
    * [cite_start]**차별성**: 플랫폼 운송 사업자가 아닌 커뮤니티 서비스로서 결제 및 배차 방식의 간소화 [cite: 70, 73]
* [cite_start]**대학생 커뮤니티(에브리타임)**: 단순 텍스트 게시판 [cite: 74, 75]
    * [cite_start]**차별성**: 매칭 후 Kakao Map 기반 실시간 GPS 공유를 통한 탑승지 매칭 효율성 극대화 [cite: 79, 80, 81]

## 4. 시스템 구성도

* **구조 설명**:
    * [cite_start]사용자가 Mobile Web을 통해 접속 [cite: 111]
    * [cite_start]**AWS ELB(Load Balancer)**를 통해 트래픽을 분산 [cite: 100, 115]
    * [cite_start]**AWS EC2 Auto Scaling Group**이 트래픽 변화에 따라 서버 자원(Scale-Out)을 유연하게 처리 [cite: 51, 94, 101, 115]
    * [cite_start]**AWS RDS**에 방 정보 및 사용자 정보 저장 [cite: 103]
    * [cite_start]**Kakao API** 및 **LocalMap Mobility**를 통해 위치 정보 서비스 제공 [cite: 104, 105]

## 5. 동작 환경
* [cite_start]**클라이언트**: 별도 앱 설치가 필요 없는 스마트폰 중심의 Mobile Web (PC 브라우저 호환) [cite: 111, 112]
* [cite_start]**서버 및 DB**: AWS EC2(WAS), RDS(Database)를 활용한 클라우드 환경 구축 [cite: 113, 114]
